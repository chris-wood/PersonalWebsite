☕ My Daily Grind

I am a third-year Ph.D. candidate at UCI with a research focus on applied cryptography and network security under Gene Tsudik. I am also a core developer of CCNx in the Computer Science Laboratory at PARC (Palo Alto Research Center).

I graduated with a B.S. and M.S. in Computer Science and B.S. in Software Engineering from RIT in 2013. During my time there I was advised by Dr. Stanislaw Radziszowski. In the past, I worked at Intel, L-3 Communications, and other small software firms. I received the NSF Graduate Research Fellowship (NSF GRFP) in 2014. I am also a member of the IEEE, ACM, IACR, SIAM, Internet Society, and Tau Beta Pi organizations.

💻 Software

All of this code is work in progress and was written either to help my research or learn some new technology.

  1. COOL (COntent Oriented Lisp): an experimental programming language built with native CCNx support.
  2. iris: a prototypical CCN forwarder written in Rust.
  3. ccnx-utils: A suite of tools for CCN research and development. The project includes a packet generator written in Haskell, C library to dissect wire-format messages, C programs to stress test a forwarder, Python program to transform DOT topologies to forward configuration files, and a set of scripts to help aid forwarder experimentation.
  4. protoccn: A framework and DSL for developing CCN applications in Python.
  5. flux: A language for programmable packet flows.
  6. enfs: An encrypted network filesystem based on CCN and FUSE.
  7. Threatspec: code-driven threat modeling.
  8. libforte: the early start of a usable cryptography and security library written in C.
  9. check: A tool to measure development productivity.
  10. papersmith: A utility to enable better writing with Markdown. It compiles Markdown to LaTeX. Scripts for spelling and grammar checks are provided.
